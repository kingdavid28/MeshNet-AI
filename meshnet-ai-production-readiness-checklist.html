! Rotate VITE_MESH_SECRET / MESH_SECRET — the dev key is already public

The 128-char hex secret currently in .env.local is committed to git history. Generate a fresh one:

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Set it in both .env.local (VITE_MESH_SECRET) and backend/config/.env (MESH_SECRET). Never commit either file again.

2 · Frontend Build

! Run a production Vite build and serve via nginx, not vite preview

The docker-compose file serves the frontend with vite preview — that is a local-preview tool, not a production server. The compose file itself warns this:

# 1. Build the static bundle
pnpm build          # outputs to dist/

# 2. Serve with nginx (replace the frontend service in docker-compose.yml)
# Dockerfile.frontend.prod
FROM nginx:alpine
COPY dist/ /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

A minimal nginx.conf for a React SPA:

server {
  listen 80;
  root /usr/share/nginx/html;
  index index.html;
  location / { try_files $uri $uri/ /index.html; }
  location /api/ { proxy_pass http://express:4000; }
}
frontenddocker
! Set production environment variables at build time

Vite bakes VITE_* vars into the JS bundle at build time. For production, supply a proper .env.production:

VITE_API_BASE_URL=https://api.yourdomain.com
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJ...
VITE_MESH_SECRET=<rotated-secret>
VITE_SIM_MODE=false

Never commit this file. Use CI/CD secrets injection instead (GitHub Actions, Railway, etc.).

3 · Backend — Express (Layer 4)

! Set NODE_ENV=production in backend/config/.env

Express disables stack traces in error responses and enables various optimisations when NODE_ENV=production. Without it, internal errors can leak stack traces to clients.

NODE_ENV=production
expresssecurity
~ Lock CORS_ORIGINS to your actual domain

The Python FastAPI defaults to allowing localhost:5173, localhost:4173, and localhost:4000 when CORS_ORIGINS is unset. In production, set this in backend/config/.env:

CORS_ORIGINS=https://app.yourdomain.com
fastapisecurity
~ Replace hardcoded localhost URLs in graph_builder.py

MeshGraphBuilder defaults to http://localhost:4000. In Docker Compose this resolves correctly via the BACKEND_API_URL env var — but make sure every instantiation respects that env var:

# backend/config.py — verify BACKEND_API_URL is read here
# In graph_builder.py the api_base default is "http://localhost:4000"
# It must be overridden from config / env at startup in api_server.py

4 · Backend — FastAPI / Uvicorn (Layer 3)

! Run uvicorn with multiple workers behind gunicorn

The dev command uvicorn api_server:app --port 5050 is single-process. For production:

pip install gunicorn
gunicorn api_server:app \
  --worker-class uvicorn.workers.UvicornWorker \
  --workers 4 \
  --bind 0.0.0.0:5050 \
  --access-logfile - \
  --error-logfile -

Update your Dockerfile.python CMD to use the gunicorn command above.

pythonperformance
~ Disable FastAPI's interactive docs in production

FastAPI exposes /docs and /redoc publicly by default. Disable them so your internal API isn't browseable by anyone:

app = FastAPI(
    ...,
    docs_url=None,     # disable /docs
    redoc_url=None,    # disable /redoc
    openapi_url=None,  # disable /openapi.json
)

Or gate them behind DEBUG=true only.

pythonsecurity

5 · TLS / HTTPS

! Terminate TLS in front of every service — nothing should be served over HTTP in production

Options (pick one):

  • Cloudflare Tunnel — free, zero-config TLS. Best for disaster-response deployments.
  • nginx + Let's Encrypt (Certbot) — reverse proxy in front of Docker Compose on a VPS.
  • Railway / Render / Fly.io — managed platforms that provide TLS automatically.

The SSE endpoint (/api/signal/stream) also requires HTTPS; plain HTTP SSE gets killed by corporate proxies.

infrasecurity

7 · Android / Capacitor

~ Update capacitor.config.ts server URL for production

The Capacitor config likely still has a server.url pointing to localhost. For production builds targeting real devices, point it to your HTTPS domain so the PWA inside the WebView hits the real API:

// capacitor.config.ts
const config: CapacitorConfig = {
  ...
  server: {
    url: "https://app.yourdomain.com",   // ← replace
    cleartext: false,                    // never allow HTTP
  }
};

Then rebuild and resync: pnpm cap:sync

8 · Observability & Ops

~ Add structured logging and a log aggregator

Both services use console.* / Python logging. For production, ship logs to a collector:

  • Papertrail or Logtail — free tier, easy Docker syslog driver
  • Grafana Loki — self-hosted, pairs well with Prometheus

~ Wire up the existing /health and /api/health endpoints to an uptime monitor

Both the Express and FastAPI services already expose health endpoints. Point an external monitor (UptimeRobot free, BetterUptime, etc.) to them so you get alerts if either service goes down.
